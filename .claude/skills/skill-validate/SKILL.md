# 스킬 검증

## 트리거 조건

- 새로운 스킬(SKILL.md)을 생성한 직후
- 기존 스킬을 수정한 직후
- `/skill-validate`로 수동 호출 (인자: 스킬 디렉토리 경로)

## 검증 체크리스트

생성/수정된 스킬의 SKILL.md와 디렉토리 구조를 읽고, 아래 항목을 모두 검증한다.
각 항목은 PASS / WARN / FAIL로 판정한다.

### 1. 프론트매터 필수 필드

| 항목 | 기준 | 등급 |
|------|------|------|
| `name` 존재 | 1-64자, 소문자/숫자/하이픈만 | FAIL |
| `name` = 디렉토리명 | 부모 디렉토리명과 일치해야 함 | FAIL |
| `description` 존재 | 1-1024자 | FAIL |
| `description` 품질 | **무엇을 하는지** + **언제 사용하는지** 모두 포함 | WARN |

### 2. SKILL.md 본문

| 항목 | 기준 | 등급 |
|------|------|------|
| 본문 길이 | 500줄 이하 | WARN |
| 과도한 설명 | 에이전트가 이미 아는 내용을 장황하게 설명하지 않는가 | WARN |
| 용어 일관성 | 같은 개념에 여러 용어를 혼용하지 않는가 | WARN |

### 3. 디렉토리 구조

| 항목 | 기준 | 등급 |
|------|------|------|
| SKILL.md 존재 | 스킬 루트에 있어야 함 | FAIL |
| 경로 컨벤션 | 윈도우 경로(`\`) 사용 금지, 슬래시(`/`)만 사용 | FAIL |
| 긴 레퍼런스 분리 | 상세 문서는 `references/` 또는 `templates/`로 분리되어 있는가 | WARN |

### 4. 토큰 효율 (Progressive Disclosure)

| 항목 | 기준 | 등급 |
|------|------|------|
| Tier 2 크기 | SKILL.md 본문이 5000토큰(~2000자) 이내 권장 | WARN |
| Tier 3 분리 | 스크립트, 레퍼런스, 에셋은 별도 파일로 분리되어 있는가 | WARN |

### 5. 안티패턴

| 항목 | 기준 | 등급 |
|------|------|------|
| 옵션 과다 | "A, B, C 중 선택" 대신 기본값을 정해서 제시하는가 | WARN |
| 시간에 민감한 정보 | 날짜, 버전 등 금방 바뀔 정보가 하드코딩되어 있지 않은가 | WARN |

### 6. CLAUDE.md 등록

| 항목 | 기준 | 등급 |
|------|------|------|
| 스킬 목록에 등록 | `.claude/CLAUDE.md`의 `## 스킬` 섹션에 항목이 추가되었는가 | WARN |

## 출력 형식

```
=== 스킬 검증: {skill-name} ===

[PASS] name 필드 유효 (skill-name, 디렉토리 일치)
[PASS] description 필드 유효 (48자)
[WARN] description에 "언제 사용하는지" 트리거 조건이 명시되지 않음
[PASS] SKILL.md 본문 120줄
[PASS] 디렉토리 구조 정상
[FAIL] CLAUDE.md 스킬 목록에 미등록

결과: PASS 4 / WARN 1 / FAIL 1
```

## 워크플로우

1. 대상 스킬의 SKILL.md를 읽는다.
2. 프론트매터를 파싱하여 필수 필드를 검증한다.
3. 디렉토리 구조를 확인한다.
4. 본문 길이와 안티패턴을 점검한다.
5. `.claude/CLAUDE.md`의 스킬 목록 등록 여부를 확인한다.
6. 결과를 출력하고, FAIL/WARN 항목에 대해 수정 제안을 한다.
7. 사용자 확인 후 자동 수정이 가능한 항목은 즉시 반영한다.
